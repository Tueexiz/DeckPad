# DeckPad — Landing page

Marketing landing page for **DeckPad** (the tablet remote-control PWA in the sibling
`client/` + `server/` folders of this repo).

Tone: French, luxe sobre. Visual register: deep indigo + platinum. Stack: Next.js 15 App
Router · TypeScript · Tailwind v4 · shadcn/ui patterns · Radix primitives · Framer Motion.

---

## 1 — Stack

| Concern | Choice |
|---|---|
| Framework | Next.js 15 (App Router, RSC, edge OG) |
| Styling | Tailwind v4 (CSS-first, `@theme` tokens) |
| Components | shadcn/ui patterns built on Radix primitives |
| Animation | Framer Motion (premium hero choreography) |
| Forms | `react-hook-form` is installed; the multi-step waitlist uses a custom typed `useReducer` state machine. `zod` validates both client and server. |
| Toasts | `sonner` |
| Email | `resend` (graceful fallback: `console.log` when `RESEND_API_KEY` is absent) |
| Analytics | Vercel Analytics |
| Hosting | Vercel (CDG region for FR latency) |

## 4 — Design system

The visual direction was generated by the `ui-ux-pro-max` skill
(`skills/ui-ux-pro-max/scripts/search.py`) and persisted to:

- `design-system/deckpad/MASTER.md` — global rules
- `design-system/deckpad/pages/landing.md` — landing-specific overrides

The skill recommended **Exaggerated Minimalism** with **Bodoni Moda / Jost** typography.
Color direction was manually shifted from the default premium-black + gold to
**deep indigo (#1E1B4B → #4F46E5) + platinum (#F5F4F0 → #A8A296)** to match the
"luxury indigo" brief.

All tokens live in `app/globals.css` under `@theme` and are consumed via CSS custom
properties (e.g. `var(--color-indigo-500)`). No raw hex values in components.

## 5 — Interaction & Logic Spec — implemented modules

| Module | State machine | Data flow | Loading / Error / Empty |
|---|---|---|---|
| `WaitlistForm` | `idle → submitting → success \| error` × `step 1→2→3` (typed `useReducer`, discriminated union in `lib/state/waitlist-machine.ts`) | `react-hook-form` not needed; controlled inputs → `zod` validation per step → `POST /api/waitlist` | Inline `aria-invalid` errors with focus on first invalid field. Submitting → spinner + `aria-busy`. Error → retry CTA + `role="alert"`. Success → animated thank-you panel + `Sonner` toast. |
| `SavingsCalculator` | Pure reducer in `lib/state/savings-machine.ts` (`SET_DEVICES` / `SET_TIER`) | Sliders + segmented control → `computeTotal()` (synchronous) → animated number via Framer Motion `animate()` | N/A (synchronous, no async, no empty case) |
| `PlatformTabs` | Radix Tabs controlled state, synced to URL hash `#plateforme=windows\|android` | Click → `setValue` → `history.replaceState` | N/A |
| `Faq` | Local search query + `useDeferredValue` → `useMemo` filter | Input change → deferred query → filtered list | Empty filter → typed empty-state copy with email fallback |
| `DemoDialog` | Radix Dialog controlled `open` | Trigger → autoplay `<video>` muted; `onError` → fallback message | `<source>` not found → fallback text panel |

## 6 — Animation system (Framer Motion)

All animations go through `components/motion/`:

- `<FadeIn>`, `<Stagger>`/`<StaggerItem>`, `<ScrollReveal>`, `<Float>` — `motion-primitives.tsx`
- `<TextReveal>` — word-by-word hero animation with blur-in
- `<AuroraBackground>` — drifting indigo radial gradients + hue-shift loop
- `<Grain>` — static decorative noise overlay
- `<MagneticCTA>` — cursor attraction with spring physics (disabled on touch + reduced-motion)
- `<LiveDeckPreview>` — 3D mouse parallax + staggered key reveal + ambient float

**Premium rules baked in everywhere:**
- Animate only `transform`, `opacity`, `filter: blur()` — never width/height/top/left
- Premium ease: `cubic-bezier(0.22, 1, 0.36, 1)`
- Spring: `{ stiffness: 120, damping: 20 }`
- `prefers-reduced-motion` collapses every primitive to instant opacity transitions
- Hero choreography sequence: aurora → navbar slide → H1 word reveal → sub → CTAs → trust → deck preview ambient

## 7 — Accessibility floor

- Contrast verified against `#07060e`: platinum-100 = 15.6:1 (AAA), indigo-500 button text = 5.7:1 (AA)
- `:focus-visible` ring is global and never removed
- Skip link to `<main>`
- Every icon-only button has `aria-label`
- Form errors use `role="alert"` and focus auto-moves to first invalid field
- All Framer Motion primitives respect `useReducedMotion()` and degrade gracefully
- FAQ Accordion uses Radix → arrow / Home / End keyboard navigation built-in
- Dialog uses Radix → focus trap + Esc to close + click-outside

## 8 — Performance

Production build output (`npm run build`):

```
Route (app)                                 Size  First Load JS
┌ ○ /                                    55.3 kB         223 kB
├ ƒ /api/waitlist                          133 B         103 kB
├ ○ /cgu, /confidentialite, /mentions-legales  165 B  159 kB
├ ƒ /opengraph-image                       133 B         103 kB
└ shared                                                102 kB
```

The 223 kB First Load is driven by Framer Motion + Radix, which is the cost of the
premium animation register requested. Optimization tactics already applied:

- `optimizePackageImports` for `lucide-react` and `framer-motion`
- `next/font` self-hosts Bodoni Moda, Jost, JetBrains Mono → no FOIT, no extra DNS
- All sections under the fold use `whileInView` (not always-on)
- Demo `<video>` is `preload="none"` and only loads when the dialog opens
- Aurora background is pure CSS animation (no JS RAF loop)

Possible future cuts: lazy-load `WaitlistForm` and `DemoDialog` via `next/dynamic`, or
ship a non-Framer fallback for the hero on slow devices.

## 9 — Deploy to Vercel

1. Push the repo, then on Vercel select the project root and set **Root Directory** to `landing/`.
2. Environment variables (in the Vercel dashboard):
   - `NEXT_PUBLIC_SITE_URL` → `https://your-domain.app`
   - `RESEND_API_KEY` → from resend.com (leave blank to keep the console fallback)
   - `RESEND_AUDIENCE_ID` → optional Resend audience to grow
   - `SUPPORT_EMAIL` → `support@your-domain.app`
   - `NOREPLY_EMAIL` → `"DeckPad <noreply@your-domain.app>"`
3. The included `vercel.json` pins the region to `cdg1` (Paris) for FR-first latency
   and adds security headers (HSTS, frame-deny, content-type sniff guard).

## 10 — What's intentionally not here

- **No CRUD admin dashboard, no auth.** DeckPad is a no-account local tool.
- **No fabricated testimonials or social logos.** Social proof relies on verifiable
  technical claims (latency, fps, network surface). A `testimonials.ts` schema is
  ready in `content/` if real quotes appear later.
- **No iOS support yet** — DeckPad's client is Android PWA only at v1.
- **No video demo file** is committed. `<DemoDialog>` will gracefully show a fallback
  message until `public/deckpad-demo.mp4` / `.webm` are added.
